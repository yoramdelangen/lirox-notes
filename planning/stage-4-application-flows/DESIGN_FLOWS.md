# Design Flows

## Design Direction

- Use `planning/stage-3-data-structures-models/DESIGN_INSPIRATION.md` as the source for visual tone and product feel.
- Convert broad principles into concrete screen behavior during flow planning.
- Keep design decisions tied to real user flows, not standalone decoration.

## Screen Flow Design

- Start from the user flow being designed.
- Identify the required screen states for that flow.
- Define the primary action, secondary actions, and empty/loading/error states.
- Check that navigation, note editing, sync state, and conflict state remain visible when needed.
- Remove UI that does not support the flow.

## Interaction Review

- Walk through the flow on desktop and mobile.
- Confirm the editor stays central and readable.
- Confirm file tree, labels, search, and sync status do not compete with writing.
- Confirm error and conflict states have clear recovery actions.

## Handoff To MVP Build

- Capture only decisions needed to build the MVP screen.
- Defer polish that does not affect usability.
- Move unresolved design gaps to Stage 5 review.
