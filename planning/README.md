# LiroxNotes Planning

This folder contains the staged planning and delivery notes for LiroxNotes.

## Stages

- `stage-1-functional-boundaries/` defines what the application should do.
- `stage-2-technical-choices/` defines programming languages, dependencies, runtime choices, and technical tradeoffs.
- `stage-3-data-structures-models/` defines application models, data structures, naming conventions, and domain definitions.
- `stage-4-application-flows/` defines user flows, sync flows, conflict flows, and important application behavior sequences.
- `stage-5-confirmation-review/` confirms that everything is connected, defined, and not missing important pieces.
- `stage-6-building-mvp/` tracks MVP implementation planning and build progress.
- `stage-7-release-first-version/` tracks release preparation for the first version.

## Working Style

- Stage 1 answers what the application should do.
- Stage 2 answers what technology should be used to build it.
- Stage 3 answers how the application concepts should be named, modeled, and structured.
- Stage 4 answers how users and systems move through the application.
- Stage 5 checks for gaps, contradictions, and missing connections.
- Stage 6 turns the plan into a working MVP.
- Stage 7 prepares the first usable release.
