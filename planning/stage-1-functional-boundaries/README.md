# Stage 1: Functional Application Boundaries

This stage defines what LiroxNotes should do functionally.

## Files

- `agents/product-agent.md` contains product direction and core application ideas.
- `agents/workspace-agent.md` contains workspace and repository decisions.
- `agents/notes-agent.md` contains note, title, frontmatter, config, attachment, trash, and filename decisions.
- `agents/labels-links-agent.md` contains label, tag, and link decisions.
- `agents/sync-agent.md` contains Git gateway, sync, offline, and conflict decisions.
- `agents/editor-agent.md` contains editor behavior decisions.
- `UKNOWNS.md` contains Stage 1 loose ends and resolved functional decisions.
